# SRT Audio Streamer - テストガイド

このドキュメントでは、SRT Audio Streamerアプリの包括的なテスト手順を説明します。

## 事前準備

### 必要な機材・環境

1. **開発用Mac**: FFmpegまたはOBS Studioがインストール済み
2. **iOSデバイス**: iPhone（iOS 17.0以上）
3. **ネットワーク**: MacとiPhoneが同じWi-Fiネットワークに接続
4. **Xcode**: バージョン15.0以上

### テストサーバーのセットアップ

#### FFmpegを使用する場合

```bash
# インストール（初回のみ）
brew install ffmpeg

# SRT受信サーバーを起動
ffmpeg -i srt://0.0.0.0:9710?mode=listener -f null -

# 詳細なログを表示する場合
ffmpeg -v verbose -i srt://0.0.0.0:9710?mode=listener -f null -
```

#### OBS Studioを使用する場合

1. OBS Studioを起動
2. 設定 → 配信
3. サービス: カスタム
4. サーバー: `srt://0.0.0.0:9710?mode=listener`
5. OK → 配信開始

### MacのIPアドレス確認

```bash
# 方法1: ifconfig
ifconfig | grep "inet " | grep -v 127.0.0.1

# 方法2: システム環境設定
# システム環境設定 → ネットワーク → Wi-Fi → 詳細
```

例: `192.168.1.10`

## テストケース

### 1. 初期セットアップテスト

#### 1.1 初回起動とマイク権限

**目的**: マイク権限が正しく要求されることを確認

**手順**:
1. アプリを初めて起動
2. 配信開始ボタンをタップ

**期待結果**:
- マイク権限のダイアログが表示される
- 「許可」を選択すると配信が開始される
- 「許可しない」を選択するとエラーメッセージが表示される

**確認項目**:
- [ ] マイク権限ダイアログが表示される
- [ ] 許可後に配信が開始される
- [ ] 拒否後にエラーメッセージ「マイクへのアクセスが許可されていません」が表示される

#### 1.2 UI初期状態

**目的**: UI要素が正しく初期化されることを確認

**手順**:
1. アプリを起動（マイク権限は許可済み）

**期待結果**:
- 状態インジケーターがグレー（待機中）
- SRT URL入力フィールドが空
- ビットレート選択が64 kbpsに設定
- 配信開始ボタンが表示される
- ビットレート表示はなし

**確認項目**:
- [ ] 初期状態が「待機中」
- [ ] 状態インジケーターがグレー
- [ ] デフォルトビットレートが64 kbps

---

### 2. 基本配信機能テスト

#### 2.1 正常な配信開始・停止

**目的**: 基本的な配信フローが正常に動作することを確認

**手順**:
1. SRT URLを入力: `srt://[MacのIP]:9710?mode=caller`
2. ビットレートは64 kbpsのまま
3. 「配信開始」ボタンをタップ
4. 10秒間待機
5. 「配信停止」ボタンをタップ

**期待結果**:
- 配信開始後、状態が「接続中」→「配信中」に変化
- ビットレート表示が更新される（約64.0 kbps）
- FFmpeg/OBSで音声が受信される
- 配信停止後、状態が「切断中」→「待機中」に変化

**確認項目**:
- [ ] 状態遷移: 待機中 → 接続中 → 配信中
- [ ] 状態インジケーターの色変化: グレー → オレンジ → グリーン
- [ ] ビットレート表示が表示される
- [ ] ビットレートの値が更新される
- [ ] FFmpeg/OBSで音声データが受信される
- [ ] 停止後の状態遷移: 配信中 → 切断中 → 待機中
- [ ] 停止後、ビットレート表示が消える

**FFmpegでの確認方法**:
```
# FFmpegの出力で以下のようなログが表示されるはず
Stream #0:0: Audio: aac, 44100 Hz, stereo, fltp, 64 kb/s
```

#### 2.2 音声の送受信確認

**目的**: 実際の音声が正しく送信されることを確認

**手順**:
1. 配信を開始
2. iPhoneのマイクに向かって話す
3. FFmpegのログを確認

**期待結果**:
- FFmpegのログでオーディオストリームが検出される
- マイクから音声入力があることが確認できる

**確認項目**:
- [ ] FFmpegでAudioストリームが認識される
- [ ] サンプルレートが44100 Hzである
- [ ] チャンネル数が正しい（mono/stereo）

---

### 3. ビットレート変更テスト

#### 3.1 ビットレート選択

**目的**: ビットレートの変更が正しく適用されることを確認

**テストケース1: 64 kbps**
1. ビットレートを64 kbpsに設定
2. 配信開始
3. ビットレート表示を確認

**期待結果**: 約64.0 kbps

**テストケース2: 96 kbps**
1. 配信停止
2. ビットレートを96 kbpsに設定
3. 配信開始
4. ビットレート表示を確認

**期待結果**: 約96.0 kbps

**テストケース3: 128 kbps**
1. 配信停止
2. ビットレートを128 kbpsに設定
3. 配信開始
4. ビットレート表示を確認

**期待結果**: 約128.0 kbps

**確認項目**:
- [ ] 64 kbps設定時のビットレート表示が正しい
- [ ] 96 kbps設定時のビットレート表示が正しい
- [ ] 128 kbps設定時のビットレート表示が正しい
- [ ] 配信中はビットレート選択が無効化される

#### 3.2 配信中のビットレート変更不可

**目的**: 配信中はビットレートを変更できないことを確認

**手順**:
1. 配信を開始
2. ビットレート選択を操作しようとする

**期待結果**:
- ビットレート選択が無効化（グレーアウト）されている
- 選択を変更できない

**確認項目**:
- [ ] 配信中はビットレート選択が無効化される

---

### 4. エラーハンドリングテスト

#### 4.1 無効なURL入力

**目的**: 入力バリデーションが正しく機能することを確認

**テストケース1: 空のURL**
1. SRT URLを空のまま
2. 配信開始ボタンをタップ

**期待結果**: 配信開始ボタンが無効化されている

**テストケース2: 無効なプロトコル**
1. SRT URLに `http://192.168.1.10:9710` を入力
2. 配信開始ボタンをタップ

**期待結果**: エラーメッセージ「無効なSRT URLです」が表示される

**テストケース3: 不正なフォーマット**
1. SRT URLに `srt://` のみを入力
2. 配信開始ボタンをタップ

**期待結果**: エラーが発生する

**確認項目**:
- [ ] 空のURL時、配信開始ボタンが無効化される
- [ ] 無効なプロトコル時、エラーメッセージが表示される
- [ ] エラーメッセージが赤色で表示される
- [ ] エラーアイコンが表示される

#### 4.2 接続失敗

**目的**: サーバーに接続できない場合の挙動を確認

**手順**:
1. FFmpeg/OBSを停止
2. SRT URLを入力
3. 配信開始ボタンをタップ
4. タイムアウトまで待機

**期待結果**:
- 状態が「接続中」のまま待機
- その後エラー状態に遷移
- エラーメッセージが表示される

**確認項目**:
- [ ] 接続失敗時にエラー状態に遷移する
- [ ] エラーメッセージが表示される
- [ ] 状態インジケーターが赤色になる

#### 4.3 マイク権限拒否

**目的**: マイク権限が拒否された場合の挙動を確認

**手順**:
1. 設定アプリでマイク権限を無効化
2. アプリに戻る
3. 配信開始ボタンをタップ

**期待結果**:
- エラーメッセージ「マイクへのアクセスが許可されていません」が表示される
- 配信が開始されない

**確認項目**:
- [ ] エラーメッセージが表示される
- [ ] 状態が「エラー」になる
- [ ] 設定アプリへの誘導メッセージが表示される

#### 4.4 ネットワーク切断

**目的**: 配信中にネットワークが切断された場合の挙動を確認

**手順**:
1. 配信を開始
2. iPhoneのWi-Fiを無効化
3. 挙動を観察
4. Wi-Fiを再度有効化

**期待結果**:
- エラー状態に遷移
- エラーメッセージが表示される

**確認項目**:
- [ ] ネットワーク切断時にエラー検出される
- [ ] エラーメッセージが表示される
- [ ] 再接続が必要

### 5. UI/UX テスト

#### 5.1 状態インジケーター

**目的**: 各状態で正しい色とテキストが表示されることを確認

**テストケース**:
| 状態 | 色 | テキスト |
|------|-----|----------|
| idle | グレー | 待機中 |
| connecting | オレンジ | 接続中 |
| streaming | グリーン | 配信中 |
| error | レッド | エラー: [メッセージ] |

**確認項目**:
- [ ] 待機中: グレーの円 + 「待機中」
- [ ] 接続中: オレンジの円 + 「接続中」
- [ ] 配信中: グリーンの円 + 「配信中」
- [ ] エラー: レッドの円 + 「エラー: [メッセージ]」

#### 5.2 レスポンシブデザイン

**目的**: 様々な画面サイズで正しく表示されることを確認

**テストデバイス**:
- iPhone SE (第3世代) - 小画面
- iPhone 15 - 標準サイズ
- iPhone 15 Pro Max - 大画面

**確認項目**:
- [ ] 小画面でもUIが崩れない
- [ ] テキストが読みやすい
- [ ] ボタンがタップしやすいサイズ
- [ ] スクロールが必要な場合はスクロール可能

#### 5.3 キーボード表示

**目的**: キーボード表示時のUIが正しく動作することを確認

**手順**:
1. SRT URL入力フィールドをタップ
2. キーボードを表示
3. URLを入力
4. キーボードを閉じる

**確認項目**:
- [ ] キーボード表示時にUIが隠れない
- [ ] URLが正しく入力できる
- [ ] オートコンプリートが無効化されている
- [ ] キャピタライゼーションが無効化されている

### 6. パフォーマンステスト

#### 6.1 長時間配信

**目的**: 長時間の配信でメモリリークやパフォーマンス低下がないことを確認

**手順**:
1. 配信を開始
2. 30分間配信を継続
3. Xcodeのメモリグラフで確認

**期待結果**:
- メモリ使用量が増加し続けない
- アプリがクラッシュしない
- ビットレート表示が安定している

**確認項目**:
- [ ] 30分間クラッシュなく動作
- [ ] メモリリークがない
- [ ] CPU使用率が安定
- [ ] バッテリー消費が過度でない

#### 6.2 繰り返し開始・停止

**目的**: 配信の開始・停止を繰り返してもリソースリークがないことを確認

**手順**:
1. 配信を開始
2. 10秒後に停止
3. 5秒待機
4. 上記を10回繰り返す

**期待結果**:
- 毎回正常に開始・停止できる
- メモリ使用量が増加し続けない
- エラーが発生しない

**確認項目**:
- [ ] 10回の開始・停止が正常に動作
- [ ] メモリリークがない
- [ ] エラーが発生しない

#### 6.3 バックグラウンド動作

**目的**: バックグラウンドでも配信が継続されることを確認

**手順**:
1. 配信を開始
2. ホームボタンを押してアプリをバックグラウンドに移動
3. 1分間待機
4. アプリに戻る

**期待結果**:
- バックグラウンドでも配信が継続される
- アプリに戻ったとき、配信中のまま
- ビットレート表示が更新されている

**確認項目**:
- [ ] バックグラウンドで配信継続
- [ ] FFmpeg/OBSで音声が継続受信される
- [ ] フォアグラウンド復帰後、状態が正常

## テスト環境

### 推奨テスト環境

- **Mac**: macOS 14.0以上
- **iPhone**: iOS 17.0以上の実デバイス
- **ネットワーク**: 安定したWi-Fi（5GHz推奨）
- **Xcode**: 15.0以上

### FFmpegの詳細オプション

```bash
# 標準的な受信
ffmpeg -i srt://0.0.0.0:9710?mode=listener -f null -

# 詳細ログ付き
ffmpeg -v verbose -i srt://0.0.0.0:9710?mode=listener -f null -

# 音声ファイルとして保存
ffmpeg -i srt://0.0.0.0:9710?mode=listener output.aac

# リアルタイム再生
ffmpeg -i srt://0.0.0.0:9710?mode=listener -f audiotoolbox default
```
